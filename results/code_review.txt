[CRITICAL] Неправильное использование Playwright Test Fixtures
- Файл/Строка: Все три файла тестов (примерно строки 1-10 в каждом)
- Что не так: Вместо использования встроенных фикстур Playwright (test.beforeEach с параметром { page }), код создает новую страницу через browser.newPage() в каждом before-хуке. Это нарушает стандартную модель работы Playwright, где page управляется фреймворком. Также отсутствует закрытие браузера (browser.close()), что может привести к утечкам памяти.
- Влияние на корректность системы: Тесты не будут работать корректно в контексте Playwright, могут возникать проблемы с изоляцией тестов, утечки ресурсов, неправильная работа с контекстами.
- Как починить: Использовать стандартные фикстурные параметры Playwright:

```typescript
test.describe('Group MainPage tests @main @regression', () => {
    let mainPage: MainPage;

    test.beforeEach(async({ page }) => {
        mainPage = new MainPage(page);
        await mainPage.navigate();
    });

    // Убрать afterEach, так как page управляется Playwright
});
```

[HIGH] Дублирование импортов и объявлений test.describe
- Файл/Строка: Все три файла тестов (строки 1-5 в каждом)
- Что не так: Каждый файл содержит одинаковые импорты и структуру describe-блоков. Это нарушает принцип DRY (Don't Repeat Yourself) и усложняет поддержку.
- Влияние на корректность системы: При изменении базовой конфигурации нужно править все файлы, высокая вероятность ошибок и несогласованности.
- Как починить: Вынести общую логику в базовый класс или использовать хуки на уровне конфигурации:

```typescript
// base.test.ts
import { test as baseTest } from '@playwright/test';

export const test = baseTest.extend({
    mainPage: async ({ page }, use) => {
        const mainPage = new MainPage(page);
        await mainPage.navigate();
        await use(mainPage);
    }
});
```

[HIGH] Отсутствие проверки навигации перед действиями
- Файл/Строка: MainPage tests строка 20, TournamentsPage tests строка 15, GamePage tests строка 15
- Что не так: В тестах CA-2, CA-3, CA-4, CA-6, CA-7 отсутствует явная проверка успешной навигации перед выполнением действий (clickRegisterButton, clickTournamentsMenuLink и т.д.).
- Влияние на корректность системы: Тесты могут падать из-за race condition, если страница не успела загрузиться перед кликом.
- Как починить: Добавить ожидание загрузки страницы или использовать waitForLoadState:

```typescript
test("[CA-2] Проверка открытия попапа регистрации с главной страницы @test", async () => {
    await mainPage.navigate();
    await mainPage.page.waitForLoadState('networkidle'); // или 'domcontentloaded'
    await mainPage.clickRegisterButton();
    const signUpPopup = mainPage.page.locator('text=Sign up');
    await expect(signUpPopup).toBeVisible();
});
```

- Файл/Строка: MainPage tests строка 21, GamePage tests строки 16, 21, 30-31
- Что не так: Используются локаторы по тексту ('text=Sign up', 'text=Become a pro', 'text=Ram divakar Cherla'), которые чувствительны к изменениям UI, переводам и регистру.
- Влияние на корректность системы: Тесты будут ломаться при любом изменении текста на странице, даже если функциональность не изменилась.
- Как починить: Использовать более стабильные селекторы (data-testid, role, атрибуты):

[MEDIUM] Отсутствие типизации для возвращаемых значений
- Файл/Строка: Все файлы, методы Page Object (clickRegisterButton, navigate и т.д.)
- Что не так: В предоставленном коде не видно объявлений методов Page Object, но если они не имеют типов возвращаемых значений (Promise<void>), это может привести к ошибкам компиляции.
- Влияние на корректность системы: Потенциальные runtime ошибки, сложность отладки, отсутствие автодополнения в IDE.
- Как починить: Добавить явные типы в методах Page Object:

```typescript
// В MainPage class
async navigate(): Promise<void> {
    await this.page.goto('https://worldchess.com/');
}

async clickRegisterButton(): Promise<void> {
    await this.page.locator('[data-testid="register-btn"]').click();
}
```
